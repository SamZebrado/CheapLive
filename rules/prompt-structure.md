# 开发 Prompt 编写规范

本文件定义开发Work 为开发Code 编写 Prompt 的固定结构、粒度规则和约束边界。

## 1. Prompt 前置声明

每份 Prompt 开头必须附一段不超过十行的依据声明：

```text
Prompt 依据
- 当前 GitHub HEAD：
- 用户最新决定：
- 本轮开发Code 上一条回复：
- 已确认问题：
- 尚未确认问题：
- 本 Prompt 是否修改产品路线：否
```

## 2. Prompt 固定结构

每份给开发Code 的 Prompt 必须包含以下部分：

### 2.1 当前基线

- 仓库
- branch
- 已知 HEAD
- preflight 命令
- 远端同步要求
- 工作区清洁要求

### 2.2 本轮唯一主要目标

用一至两句话明确本轮到底交付什么。

### 2.3 已确定约束

列出不可自行改变的产品路线、兼容目标和技术边界。

### 2.4 非目标

明确本轮禁止做什么，尤其防止：

```text
顺便重构
顺便增加功能
顺便接入 Live2D
顺便修改架构
```

### 2.5 实施顺序

按依赖关系排序，避免 Agent 自行重新规划大方向。

### 2.6 技术要求

给出必要接口、文件范围、协议、安全要求和生命周期要求。不要在用户没有批准时把一种实现细节写成唯一选择；可以给首选方案和允许的替代条件。

### 2.7 自动测试与运行时验证

测试必须覆盖真实生产路径。分别列出：

- 单元测试
- 集成测试
- 浏览器 E2E
- Android 构建
- 真机待验证

每份 Prompt 不需要穷举所有可能测试，但必须明确要求开发Code：

```text
根据本轮真实改动主动设计运行时、实例化、真实入口和回归测试。
```

具体要求见 `rules/runtime-validation-gate.md`。

### 2.8 验收标准

使用可观察、可测量条件。

错误示例：

```text
优化体验
完善连接
提高稳定性
```

正确示例：

```text
receiver 收到 face-frame 后，Avatar 对应参数在 500ms 内发生变化
错误 token 的 WebSocket 连接被关闭
停止会话后旧 token 失效
```

### 2.9 完成口径

明确什么条件下可写"完成"，没有真机验证时应该如何降级表述。

### 2.10 Git 交付

要求：

- 分阶段 commit
- push
- 本地与远端 SHA 核对
- `git status --short`
- 不修改 remote、Keychain 或全局 Git 设置

### 2.11 最终汇报格式

要求列出：

- 修改文件
- 测试结果
- failed/skipped/flaky
- artifacts
- APK 或构建路径
- 未验证项
- commit
- 远端 SHA
- 工作区状态

## 3. Prompt 粒度规则

一份 Prompt 只能有一个主要工作包。

适合：

```text
搭建 Android Capture 骨架并通过 assembleDebug
打通 WebSocket 参数同步最小闭环
修复 VoiceChanger 生产链路
```

不适合：

```text
完成 Android App、音频、Live2D、悬浮窗、PWA 和全部兼容性
```

任务过大时应拆成固定 Phase，但不得要求用户在每个低风险 Phase 之间确认。

## 4. 避免过度指定

Prompt 应约束结果和关键边界，但不要无意义地指定所有实现细节。

必须指定：

- 安全边界
- 用户体验
- 协议
- 验收
- 兼容目标
- 禁止事项

可以留给开发Code 判断：

- 局部函数名
- 小范围文件拆分
- 可逆重构
- 测试辅助函数
- 不改变架构的实现细节

如果某技术选择仍不确定，应先让开发Work 调研或让开发Code 做只读 preflight，不能伪装成确定答案。

## 5. 识别环境借口

开发Code 声称"环境限制"时，必须检查：

1. 是否以前在同类环境成功过？
2. 是否只是 PATH、端口、权限或启动时序问题？
3. 是否可以使用 `process.execPath`、绝对路径或受控 fixture？
4. 是否删除测试或降低断言来绕开失败？
5. 是否有日志、SDP、ICE 状态或错误码支持判断？

未经验证不得接受：

```text
headless 一定不支持
沙箱一定不允许
GitHub Pages 不能做
浏览器不能实现
```

正确表述应区分：

```text
平台根本限制
当前架构限制
当前实现 bug
当前测试环境限制
尚未定位
```
