# 产品路线与技术边界

本文件记录已确定的产品路线、技术决策和冻结项。除非用户明确改变，不得重新讨论或改写。

## 1. 当前产品路线

### 1.1 单机模式

```
GitHub Pages → 浏览器摄像头 → MediaPipe → 程序化 Avatar
```

特征：

- 纯网页
- 零安装
- 不需要电脑
- 作为线上主 Demo

### 1.2 多端模式

```
面捕设备：CheapLive Capture Android App
直播设备：普通浏览器
```

Android App 负责：

- WebView 复用现有面捕
- 本地 HTTP/WebSocket 服务器
- 二维码和 token
- Avatar 参数发送
- 托管浏览器 receiver 页面

直播端无需安装 App，也不需要电脑。

### 1.3 Live2D

Demo 阶段冻结：

- 不继续开发
- 不下载 SDK
- 不下载或重新分发模型
- UI 隐藏或标记为后续规划
- 仅在核心 Demo 稳定、用户论文节点完成或项目入围后重新评估

## 2. 已确定技术决策

| 决策 | 内容 | 日期 |
|------|------|------|
| 纯网页技术栈 | 所有核心功能采用纯浏览器技术实现 | 2026-06-16 |

## 3. 技术边界

- 纯网页技术栈（HTML/CSS/JS）
- 优先复用现有 TransparentFloatingBrowser 悬浮能力
- 所有计算在浏览器本地完成
- 支持用户加载自定义 Live2D 模型（冻结中）

## 4. 隐私原则

- 摄像头画面默认只在本地处理
- 不上传用户面部图像
- 不保存摄像头视频
- 多端通信默认只允许局域网访问
- 配对后才能传输控制参数
- 用户可以随时关闭摄像头和局域网服务

## 5. 非目标（明确不做）

- 完整直播推流平台
- 云端账户系统
- 模型商店
- iOS 版本
- 高质量实时变声（初赛阶段）
- 完整双端视频合成（初赛阶段）

## 6. Android APP 状态变更（2026-06-20）

### 6.1 冻结声明

`android-capture/` 目录下的 Android APP 开发自 2026-06-20 起在主项目中**冻结**。

- 主项目不再更新 Android APP 代码、文档或功能
- 主项目 README 和公开页面应标注：「Android APP 功能暂停开发，比赛结束后酌情决定是否恢复」
- 现有代码保留在仓库中，但不作为活跃开发目标

### 6.2 冻结原因

Android APP 功能（手机端面捕 + 局域网传输）被纳入参赛项目独立开发。该功能：
- 尚未完成（仅完成离线演示原型，真实摄像头面捕和多端链路未实现）
- 未作为成品发布
- 符合「未公开发布的独立功能」参赛条件

### 6.3 参赛项目接管

参赛项目 `contest-entry/` 将基于主项目的 Android APP 代码继续开发：
- 参赛项目独立 git 追踪 Android APP 演进
- 参赛项目的 Android 功能不开源、不公开
- 比赛结束后，参赛项目的 Android 成果可选择性回灌主项目

### 6.4 对主项目 Agent 的指令

任何处理主项目的 Agent：
- **不得**在 `android-capture/` 目录中新增功能或修改代码
- **不得**在公开文档中将 Android APP 作为活跃功能宣传
- 如用户要求恢复 Android APP 开发，提醒用户该功能已移交参赛项目
- 如需修改 `android-capture/README.md` 等公开文件，仅允许添加「暂停开发」声明

## 6. 人工参与预算

用户当前首要任务是完成毕业论文。CheapLive 必须限制用户人工投入。

开发 Prompt 应遵循：

- 低风险、可逆、结论明确的操作由开发Code 自主完成
- 不要求用户逐项确认
- 不要求用户手动读代码或排错
- 用户只承担产品取舍、视觉验收和真机验收
- 除非阻塞核心 Demo，不产生需要用户连续操作超过 15 分钟的方案
- 单轮开发只设置一个主要目标
- 不因为功能"很酷"而扩大初赛范围

当某任务需要大量人工素材制作、许可证处理、复杂部署或长时间手动调试时，默认放入：

```
入围后 / 论文节点后 / 后续路线图
```
