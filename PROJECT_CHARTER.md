# CheapLive 项目章程

## 项目目标

为没有电脑或设备性能有限的个人主播，提供支持单端和多端组合的移动 Live2D 面捕与直播辅助工具。

核心价值：降低设备、软件、性能和学习门槛。

## 用户群体

- 使用手机进行游戏直播的个人主播
- 刚开始尝试虚拟形象直播的小主播
- 不方便真人出镜的内容创作者
- 没有直播电脑或不希望搭建复杂环境的用户
- 希望减少后台应用数量的移动主播
- 希望把旧手机重新利用为面捕设备的用户

## 核心功能

1. Live2D 模型基础显示
2. 透明悬浮显示
3. 手动表情与动作控制
4. 单端基础面捕
5. 单端性能控制
6. 双端局域网面捕
7. 双端直播制作
8. 可选实时变声

## 技术路线

- **面捕**：MediaPipe Face Landmarker（浏览器端，GPU 加速）
- **Live2D 渲染**：Live2D Cubism Web SDK（浏览器端）
- **变声**：Web Audio API + SoundTouchJS（浏览器端）
- **多端传输**：WebRTC DataChannel（局域网 P2P）
- **悬浮显示**：基于现有 TransparentFloatingBrowser 的 WebView 悬浮能力

## 非目标

- 完整直播推流平台
- 云端账户系统
- 模型商店
- iOS 版本
- 高质量实时变声（初赛阶段）
- 完整双端视频合成（初赛阶段）

## 隐私原则

- 摄像头画面默认只在本地处理
- 不上传用户面部图像
- 不保存摄像头视频
- 多端通信默认只允许局域网访问
- 配对后才能传输控制参数
- 用户可以随时关闭摄像头和局域网服务

## 技术边界

- 纯网页技术栈（HTML/CSS/JS）
- 优先复用现有 TransparentFloatingBrowser 悬浮能力
- 所有计算在浏览器本地完成
- 支持用户加载自定义 Live2D 模型

## 发布要求

- 开源依赖必须记录许可证
- 用户导入的模型必须提示自行确认授权
- 隐私说明必须完整
- 参赛演示必须包含单端和双端模式
